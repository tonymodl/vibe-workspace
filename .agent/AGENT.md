# КАК РАБОТАТЬ
1. При открытии проекта читай все файлы в .tasks/ со статусом READY_FOR_ANTIGRAVITY
2. Начинай с наименьшего номера TASK
3. **ПЕРЕД РАБОТОЙ** прочитай спецификации из docs/specs/ — там архитектура и правила
4. Создай ветку: feature/task-[номер]-[slug]
5. Реализуй точно по Acceptance Criteria
6. После работы обнови статус TASK файла на DONE_BY_ANTIGRAVITY
7. Создай Pull Request в main

# GIT ПРАВИЛА (ОБЯЗАТЕЛЬНО)
- НИКОГДА не пушить напрямую в main
- Каждая задача = отдельная ветка feature/*
- Формат коммита: feat(task-001): описание
- Коммить после каждого логического блока, не копи всё в один коммит
- После UI — запусти браузер-агент, сделай скриншот, приложи к PR

# ПРАВИЛА ПОРТАТИВНОСТИ (КРИТИЧНО)
Проект должен переноситься на другой компьютер/диск за 10 минут.

1. **ЗАПРЕЩЕНЫ абсолютные пути** — никаких `/home/...`, `/mnt/...`, `C:\\...` в коде
2. **ВСЕ пути** к данным через `src/lib/config.ts` → `process.env`
3. **ВСЕ API endpoints** сервисов (ChromaDB, Ollama, pyannote) через `.env`
4. **data/** — единственная директория персистентных данных, целиком в `.gitignore`
5. **docker-compose.yml** для всех внешних сервисов с портами через `.env`
6. **Новая переменная окружения** = обновить `.env.example` с комментарием
7. **Тесты** работают с изолированными данными (`DATA_DIR=./test-data`)
8. Перенос сервера = `git clone` + скопировать `data/` + `.env` + `npm install` + `docker compose up`
