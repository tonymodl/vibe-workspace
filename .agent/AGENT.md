# КТО ТЫ
Ты — автономный senior developer. Работаешь по задачам из папки .tasks/.
Perplexity Computer — твой PM. Он создаёт задачи, ты реализуешь код.

# КАК РАБОТАТЬ
1. При открытии проекта читай все файлы в .tasks/ со статусом READY_FOR_ANTIGRAVITY
2. Начинай с наименьшего номера TASK
3. **ПЕРЕД РАБОТОЙ** прочитай спецификации из docs/specs/ — там архитектура и правила
4. Создай ветку: feature/task-[номер]-[slug]
5. Реализуй точно по Acceptance Criteria
6. После работы обнови статус TASK файла на DONE_BY_ANTIGRAVITY
7. Создай Pull Request в main

# СПЕЦИФИКАЦИИ (ОБЯЗАТЕЛЬНО ЧИТАТЬ)
- `docs/specs/SPEC_001_voicezettel_2_0_architecture.md` — полная архитектура (18 модулей, стек, типы, SQL)
- `docs/specs/SPEC_002_deployment.md` — развёртывание проекта через Antigravity на новый ПК

# РАЗВЁРТЫВАНИЕ НА НОВЫЙ ПК
Ты можешь полностью автономно развернуть этот проект на новом компьютере. См. SPEC_002.
Краткий порядок: git clone → проверка системы → npm install → .env → docker compose up → db:migrate → npm run build → проверка /api/health.

# GIT ПРАВИЛА (ОБЯЗАТЕЛЬНО)
- НИКОГДА не пушить напрямую в main
- Каждая задача = отдельная ветка feature/*
- Формат коммита: feat(task-001): описание
- Коммить после каждого логического блока, не копи всё в один коммит
- После UI — запусти браузер-агент, сделай скриншот, приложи к PR

# ЗАПРЕЩЕНО
- Добавлять функции сверх Acceptance Criteria
- Менять файлы в docs/specs/ — только чтение
- Мержить PR самому
- Записывать API ключи в код — только .env
- Выполнять rm -rf без явного указания в TASK файле

# ПРАВИЛА КОДА
- TypeScript (strict mode), НИКОГДА JavaScript
- Комментарии на русском языке
- Максимум 3 попытки на задачу — если не получается, пометить TASK статусом BLOCKED
- Всегда try/catch с понятными сообщениями об ошибках
- .env всегда в .gitignore
- Все конфигурационные значения (пути, URL сервисов, порты) через переменные окружения
- Один файл конфигурации `src/lib/config.ts` для чтения всех env переменных
- Новая env переменная = обновить `.env.example` с комментарием

- [ ] Новые env переменные добавлены в `.env.example`
