## Контекст
Режим петлицы — для деловых встреч. Распознаёт несколько говорящих,
понимает кто говорит, запоминает голоса, подписывает имена.
3 состояния: Запись / Справочник / Участник.

## Задача
1. Deepgram интеграция с diarization:
   - Streaming STT + `diarize=true`
   - Каждое слово получает speaker_id
   - Группировка в фразы по speaker_id
2. Person Identification:
   - Голосовые отпечатки → ChromaDB voice_prints
   - Контекстное извлечение имён ("Давай, Миша..." → speaker_1 = Миша)
   - Когда имя узнали → обновить все предыдущие облачки
3. UI облачки транскрипции:
   - Справа от сферы
   - Каждое облачко: имя говорящего + текст
   - Уникальный цвет для каждого говорящего (владелец = синий)
4. 3 состояния ассистента:
   - Запись: только транскрибация
   - Справочник: + отвечает на обращение по имени
   - Участник: + активно участвует, заполняет паузы >3с
5. Переключатель под сферой: 3 кнопки с иконками
6. Уникальная анимация сферы:
   - Базовое: медленное дыхание, приглушённые тона
   - На голос владельца: синий trail
   - На голоса других: уникальные цвета, отдельные потоки частиц
   - Несколько говорящих: разноцветные потоки не смешиваются

## Acceptance Criteria
- [ ] Deepgram diarization работает в streaming
- [ ] Облачки справа подписаны именами или Speaker N
- [ ] Цвета облачек уникальны по говорящим
- [ ] 3 состояния переключаются кнопкой под сферой
- [ ] В состоянии "Справочник" — реагирует на имя
- [ ] В состоянии "Участник" — заполняет паузы >3с
- [ ] Сфера: разные цвета для разных говорящих
- [ ] Анимация отличается от обычного режима

## Стек
Deepgram SDK, ChromaDB, Three.js, Zustand
