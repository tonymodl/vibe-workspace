# TASK_011: Адаптивный системный промпт — вечные директивы
**Статус:** READY_FOR_ANTIGRAVITY
**Ветка:** feature/task-011-adaptive-prompt
**Спецификация:** docs/specs/SPEC_001_voicezettel_2_0_architecture.md — Модуль 4.1

## Контекст
КРИТИЧНАЯ ФИЧА: поправки пользователя к поведению ИИ сохраняются НАВСЕГДА. Если пользователь сказал "не называй меня на вы" — это запоминается навечно. Противоположная команда деактивирует старую.

## Задача
1. LLM Classifier для детекции директив:
   - Анализирует каждую фразу пользователя
   - Определяет: это поправка к поведению? (да/нет, confidence)
   - Если да → извлекает суть директивы
   - Если противоположная существующей → деактивирует старую
   - Использует дешёвую/быструю модель (Groq) чтобы не тормозить
2. CRUD для директив:
   - Создание при детекции
   - Деактивация при противоположной команде
   - Список активных директив (для UI настроек)
   - Ручное управление через UI (удалить/деактивировать)
3. Сборка промпта:
   - Базовый системный промпт (из настроек)
   - + Все активные директивы пользователя
   - + RAG контекст (из TASK_010)
   - + История текущей сессии

## Файлы для создания/изменения
- `src/lib/directives/classifier.ts` — LLM classifier для детекции директив
- `src/lib/directives/manager.ts` — CRUD директив (SQLite)
- `src/lib/rag/prompt-assembler.ts` — сборка финального промпта
- `src/app/api/directives/route.ts` — API для UI управления
- `src/components/settings/DirectivesList.tsx` — UI: список директив

## Acceptance Criteria
- [ ] "Не называй меня на вы" → сохраняется как директива, ИИ следует ей
- [ ] "Можешь на вы" → деактивирует предыдущую директиву
- [ ] Директивы переживают перезагрузку (SQLite)
- [ ] Classifier работает параллельно (не блокирует основной ответ)
- [ ] UI показывает список активных директив с возможностью удаления
- [ ] Промпт корректно собирается: base + directives + RAG + history
- [ ] Source директивы сохраняется (voice/text/settings)
- [ ] Unit тесты: classifier detection, opposite directive, prompt assembly

## Стек
